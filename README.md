# Simple Site Gulp - Проєкт з Gulp

Простий веб-сайт з автоматизованою збіркою за допомогою Gulp.

## 📋 Опис

Цей проєкт використовує Gulp для автоматизації процесу розробки. Gulp забезпечує:

- ✅ Компіляцію SCSS/SASS у CSS
- ✅ Автоматичне додавання вендорних префіксів
- ✅ Мінімізацію CSS для продакшн збірки
- ✅ Генерацію sourcemaps для зручного дебагу
- ✅ Live reload при зміні файлів (Browser Sync)
- ✅ Форматування та оптимізацію CSS коду

## 🛠 Вимоги

Перед початком роботи переконайтеся, що у вас встановлено:

- **Node.js** версії 14 або вище ([завантажити Node.js](https://nodejs.org/))
- **npm** (встановлюється разом з Node.js)

Щоб перевірити встановлені версії, виконайте в терміналі:

```bash
node --version
npm --version
```

## 📦 Встановлення

1. **Клонуйте репозиторій** (або завантажте архів):

```bash
git clone https://github.com/efftushkin/HillelFullStack-Figma-SimpleSite-Gulp.git
cd HillelFullStack-Figma-SimpleSite-Gulp
```

2. **Встановіть залежності**:

```bash
npm install
```

Ця команда встановить всі необхідні пакети, зазначені в `package.json`.

## 🚀 Запуск проєкту

### Режим розробки (Development)

Для запуску проєкту в режимі розробки з автоматичним оновленням браузера:

```bash
npm run dev
```

або просто:

```bash
npm start
```

Ця команда:
- Компілює SCSS файли в CSS
- Запускає локальний сервер на `http://localhost:3000`
- Відкриває проєкт у браузері
- Автоматично перезавантажує сторінку при зміні HTML
- Автоматично оновлює стилі без перезавантаження при зміні SCSS

### Збірка для продакшн (Production)

Для створення оптимізованої збірки:

```bash
npm run build
```

Ця команда створює:
- `css/styles.css` - розширений CSS з sourcemaps для розробки
- `css/styles.min.css` - мінімізований CSS для продакшн

## 📝 Доступні команди

| Команда | Опис |
|---------|------|
| `npm start` | Запуск режиму розробки (за замовчуванням) |
| `npm run dev` | Запуск режиму розробки з live reload |
| `npm run build` | Повна збірка для продакшн |
| `npm run styles` | Компіляція SCSS в CSS (без мінімізації) |
| `npm run minify` | Мінімізація CSS |

## 🔧 Налаштування Gulp

Конфігурація Gulp знаходиться у файлі `gulpfile.js`. Основні таски:

- **styles** - компіляція SCSS з автопрефіксами (PostCSS + Autoprefixer) та sourcemaps
- **minify** - мінімізація CSS для продакшн з додатковою оптимізацією
- **serve** - запуск локального сервера (Browser Sync) з live reload
- **dev** - режим розробки (styles + serve)
- **build** - повна збірка (styles + minify)

### Використовувані технології збірки:

- **Gulp 5.0** - система автоматизації
- **Sass (Dart Sass)** - препроцесор для SCSS/SASS
- **PostCSS + Autoprefixer** - автоматичні вендорні префікси
- **gulp-clean-css** - мінімізація та оптимізація CSS
- **gulp-sourcemaps** - генерація source maps
- **Browser Sync 3.0** - локальний сервер з live reload

## 🎨 Робота зі стилями

Всі стилі знаходяться в папці `scss/`. Для додавання нових стилів:

1. Створіть або відредагуйте файли в відповідних підпапках
2. Імпортуйте їх у `scss/styles.sass`
3. Зміни автоматично відобразяться в браузері (в режимі dev)

Gulp автоматично:
- Компілює SASS/SCSS у CSS
- Додає вендорні префікси для кращої сумісності
- Створює sourcemaps для зручного дебагу
- Мінімізує код для продакшн збірки

## 🌐 Підтримка браузерів

Автопрефікси додаються автоматично через PostCSS + Autoprefixer згідно з налаштуваннями у `.browserslistrc`:

```
last 2 versions
> 1%
ie >= 11
```

Це означає підтримку:
- **Останніх 2 версій** кожного основного браузера
- Браузерів з **часткою > 1%** світового ринку
- **Internet Explorer 11** та вище

Приклади автоматично доданих префіксів:
- `-webkit-box-sizing` для Safari/Chrome
- `-ms-flexbox` для Internet Explorer
- `-webkit-transform` для старих версій браузерів

## 🐛 Усунення неполадок

### Проблема: `npm install` завершується з помилками
**Рішення:** Переконайтеся, що використовуєте Node.js версії 14 або вище. Спробуйте видалити `node_modules` та `package-lock.json`, потім повторно виконайте `npm install`.

### Проблема: Gulp не компілює SCSS
**Рішення:** Перевірте, чи не має синтаксичних помилок у ваших SASS/SCSS файлах. Gulp виведе помилку в консоль.

### Проблема: Browser Sync не оновлює сторінку
**Рішення:** Переконайтеся, що порт 3000 не зайнятий іншою програмою. Можете змінити порт у `gulpfile.js` (рядок 69).
